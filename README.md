## 🗺️ 核心流程图

### 1. Agent 整体工作流程

```
用户输入问题
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│                    LangGraph 状态图                          │
│                                                              │
│  ┌──────────┐                                               │
│  │  START   │  ← 用户输入（比如："北京天气怎么样？"）         │
│  └────┬─────┘                                               │
│       │                                                      │
│       ▼                                                      │
│  ┌──────────┐                                               │
│  │  agent   │  ← AI 思考：我需要工具吗？                     │
│  │  节点    │                                               │
│  └────┬─────┘                                               │
│       │                                                      │
│       ├─── 需要工具 ──→ ┌──────────┐                        │
│       │                │  tools   │  ← 执行工具（查天气）    │
│       │                │  节点    │                         │
│       │                └────┬─────┘                         │
│       │                     │                               │
│       │                     └──── 返回结果 ──→ agent 节点   │
│       │                                                      │
│       └─── 不需要工具 ──→ END（直接回答用户）                │
│                                                              │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
AI 回答用户
（同时自动保存到 Checkpoint）
```

### 2. Checkpoint 记忆机制

```
第 1 次对话                    第 2 次对话
─────────────────              ─────────────────
用户："你好"                   用户："我叫什么？"
     │                              │
     ▼                              ▼
AI 回答："你好！"              AI 查询 Checkpoint
     │                              │
     ▼                              ▼
💾 保存到 Checkpoint           读取历史记录
   thread_id: "session-1"          │
   messages: [你好, 你好！]         ▼
                               AI 回答："你叫小明！"
                               （因为记得之前聊过）
```

### 3. Memory 记忆管理策略

```
┌─────────────────────────────────────────────────────────────┐
│                     三种记忆策略                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  🪟 WindowMemory（滑动窗口）                                 │
│  ┌────────────────────────────────────┐                     │
│  │ [旧消息] [消息1] [消息2] ... [消息N] │                     │
│  │              ↑────── 只保留最近 N 条 ──────↑             │
│  └────────────────────────────────────┘                     │
│                                                              │
│  📝 SummaryMemory（摘要压缩）                                │
│  ┌────────────────────────────────────┐                     │
│  │ [摘要] + [最近消息]                  │                     │
│  │   ↑                                   │                    │
│  │   └── 旧消息被压缩成摘要              │                    │
│  └────────────────────────────────────┘                     │
│                                                              │
│  🏷️ EntityMemory（实体记忆）                                 │
│  ┌────────────────────────────────────┐                     │
│  │ 记住关键信息：                       │                     │
│  │ - 用户名：张三                       │                     │
│  │ - 地点：北京                         │                     │
│  │ - 偏好：喜欢晴天                     │                     │
│  └────────────────────────────────────┘                     │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 4. Interrupt 中断点流程

```
┌─────────────────────────────────────────────────────────────┐
│                   中断点工作流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  用户请求：发送邮件给 boss@company.com                        │
│       │                                                      │
│       ▼                                                      │
│  AI 准备发送...                                              │
│       │                                                      │
│       ▼                                                      │
│  ⏸️ 触发中断！                                               │
│  ┌────────────────────────────────────┐                     │
│  │ ⚠️ 即将执行敏感操作：               │                     │
│  │ 收件人：boss@company.com            │                     │
│  │ 主题：项目进度报告                   │                     │
│  │                                    │                     │
│  │ 是否继续？(y/n)                     │                     │
│  └────────────────────────────────────┘                     │
│       │                                                      │
│       ├─── 用户输入 y ───┐                                   │
│       │                  │                                   │
│       ▼                  ▼                                   │
│  继续执行发送         取消操作                               │
│       │                  │                                   │
│       ▼                  ▼                                   │
│  ✅ 邮件已发送        ❌ 操作已取消                           │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 5. Subgraph 子图架构

```
┌─────────────────────────────────────────────────────────────┐
│                      主图（项目经理）                         │
│                                                              │
│   START ─→ research节点 ─→ writing节点 ─→ END               │
│                │                │                            │
│                ▼                ▼                            │
│         ┌──────────┐     ┌──────────┐                       │
│         │ 研究子图  │     │ 写作子图  │                       │
│         │ (车间A)  │     │ (车间B)  │                       │
│         ├──────────┤     ├──────────┤                       │
│         │ 搜索     │     │ 写草稿   │                       │
│         │   ↓      │     │   ↓      │                       │
│         │ 整理     │     │ 润色     │                       │
│         └──────────┘     └──────────┘                       │
│                                                              │
│  【好处】                                                    │
│  - 每个 子图 专注一件事                                      │
│  - 可以复用（写作子图可以被多个主图使用）                     │
│  - 易于测试（单独测试每个 子图）                              │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 6. 完整执行流程（整合所有功能）

```
用户："帮我写一篇关于 AI Agent 的文章"
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 加载记忆（Checkpoint + Memory）                     │
│  ─────────────────────────────────────────                  │
│  - 从 Checkpoint 加载会话历史                                │
│  - 用 WindowMemory 裁剪到最近 10 条消息                      │
│  - 注入 EntityMemory 中的已知实体                            │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 2: 主图分配任务                                        │
│  ──────────────────────────                                  │
│  主图决定：先研究，再写作                                     │
│  调用 research 节点                                          │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 3: 研究子图执行                                        │
│  ────────────────────────                                    │
│  搜索节点：搜索 "AI Agent" 相关内容                          │
│  整理节点：把搜索结果整理成摘要                               │
│  返回研究结果给主图                                          │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 4: 写作子图执行                                        │
│  ────────────────────────                                    │
│  草稿节点：根据研究结果写草稿                                  │
│  润色节点：优化文章语言                                       │
│  返回文章给主图                                              │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 5: 中断检查（如果需要）                                │
│  ─────────────────────────────────                           │
│  如果文章包含敏感内容 → 触发中断，等待人工审核                │
│  用户确认后继续                                              │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 6: 保存记忆                                            │
│  ─────────────────────                                       │
│  - 保存到 Checkpoint                                         │
│  - 提取实体到 EntityMemory                                   │
│  - 裁剪历史消息                                              │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
返回文章给用户 ✅
```

## 🚀 快速开始

### 1. 安装依赖

```bash
npm install
```

### 2. 配置环境变量

```bash
cp .env.example .env
# 编辑 .env，填入你的 OpenAI API Key
```

### 3. 运行

```bash
npm start
```

## 💡 核心概念速查

| 概念 | 一句话解释 | 类比 |
|------|----------|------|
| **StateGraph** | AI 的工作流程图 | 工厂流水线 |
| **Checkpoint** | 自动保存进度 | 游戏存档 |
| **Memory** | 管理对话记忆 | 大脑记忆 |
| **Interrupt** | 暂停等待确认 | BOSS 战确认 |
| **Subgraph** | 独立的小功能模块 | 工厂车间 |
| **threadId** | 会话身份证 | 身份证号 |
